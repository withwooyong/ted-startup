# 스프린트 플랜 — v1.0 MVP (6주, 3 스프린트)

## Sprint 1 (Week 1~2): 데이터 파이프라인

### 목표
KRX 데이터 수집부터 DB 적재까지의 전체 파이프라인 구축

### 백엔드
- [ ] Spring Boot 프로젝트 초기화 (Hexagonal 구조)
- [ ] PostgreSQL 스키마 설계 + 시계열 파티셔닝
- [ ] KRX 공매도 통계 크롤러 구현
- [ ] KRX 시세 데이터 크롤러 구현
- [ ] Spring Batch Job 설정 (일 1회 스케줄링)
- [ ] 배치 실패 재시도 + 알림

### 프론트엔드
- [ ] Next.js 프로젝트 초기화 (App Router)
- [ ] 레이아웃 + 네비게이션 구조

### 완료 기준
- 배치 실행 시 전 종목 데이터 수집 + DB 적재 성공
- 수집 소요시간 < 10분

---

## Sprint 2 (Week 3~4): 시그널 엔진 + 대시보드

### 목표
3대 시그널 탐지 엔진과 기본 대시보드 구축

### 백엔드
- [ ] 대차잔고 급감 시그널 로직
- [ ] 추세전환 시그널 로직 (이동평균 교차)
- [ ] 숏스퀴즈 종합 스코어링 로직
- [ ] 시그널 결과 API (GET /api/signals)
- [ ] 종목별 시계열 데이터 API (GET /api/stocks/{code}/history)

### 프론트엔드
- [ ] 시그널 리스트 페이지 (메인 대시보드)
- [ ] 종목 상세 페이지 (대차잔고 + 주가 듀얼 차트)
- [ ] Recharts 차트 컴포넌트 구현
- [ ] 면책 고지 컴포넌트 (전역)

### 완료 기준
- 3대 시그널이 정상 탐지되고 대시보드에 표시
- 차트에서 시그널 발생 시점 마커 확인 가능

---

## Sprint 3 (Week 5~6): 백테스팅 + 알림 + 마무리

### 목표
백테스팅 엔진, 텔레그램 알림, 전체 통합 테스트

### 백엔드
- [ ] 백테스팅 엔진 구현 (과거 3년)
- [ ] 백테스팅 결과 API (GET /api/backtest)
- [ ] 텔레그램 Bot API 연동
- [ ] 텔레그램 알림 스케줄링 (08:30 발송)
- [ ] 통합 테스트 (Testcontainers)

### 프론트엔드
- [ ] 백테스팅 결과 페이지 (적중률/수익률 통계)
- [ ] 텔레그램 연동 설정 페이지
- [ ] 반응형 대응 (모바일 뷰)
- [ ] 전체 접근성 검수

### 완료 기준
- 백테스팅 리포트 정상 생성 (3년 데이터)
- 텔레그램 알림 정상 발송
- 전체 기능 E2E 테스트 통과
- 라인 커버리지 80% 이상
