## 1. 프로젝트 개요

한국 주식시장(KOSPI/KOSDAQ)에서 대차잔고 급감, 추세전환, 숏스퀴즈 패턴을 분석하여
상승 가능성이 높은 종목을 탐지하는 **개인 투자 참고 도구**.

- 투자 자문 아님, 정보 제공 도구
- KRX 공개 데이터만 사용
- 모든 시그널에 면책 고지 필수

## 3. 핵심 기능 (v1 MVP)

### 3.1 데이터 수집 배치
- KRX 공매도 통계 (일별 공매도 거래대금, 대차잔고)
- KRX 시세 정보 (종가, 거래량, 시가총액)
- 일 1회 배치, 전 종목(~2,500개) 대상
- 실패 시 자동 재시도 3회 + 텔레그램 알림

### 3.2 시그널 엔진 (3대 시그널)

**시그널 1: 대차잔고 급감**
- 전일 대비 대차잔고 감소율 기준 (기본 -10%)
- 감소량, 연속 감소일수 부가 정보

**시그널 2: 추세전환**
- 대차잔고 5일/20일 이동평균 교차 (골든/데드크로스)
- 전환 강도 스코어 (약/중/강)

**시그널 3: 숏스퀴즈 종합 스코어 (0~100)**
- 대차잔고 감소율 (30%)
- 거래량 급증 비율 (25%)
- 주가 상승률 (25%)
- 공매도 비율 대비 잔고 (20%)
- 등급: A(80+) / B(60~79) / C(40~59) / D(0~39)

### 3.3 백테스팅 엔진
- 과거 3년 데이터 기반
- 시그널 타입별 적중률, 평균 수익률
- 보유 기간(5/10/20일)별 비교
- Walk-forward 검증으로 과적합 방지

### 3.4 대시보드
- 시그널 리스트 (스코어 순 정렬)
- 종목 상세 (대차잔고 + 주가 듀얼 축 차트)
- 백테스팅 결과 페이지
- 면책 고지 (모든 페이지 하단)

### 3.5 텔레그램 알림
- 일일 시그널 요약 (08:30)
- 긴급 시그널 (스코어 90+ 즉시)
- 주간 성과 리포트 (토요일 10:00)
- 배치 실패 알림

## 4. 스프린트 계획 (6주)

### Sprint 1 (Week 1~2): 데이터 파이프라인
- Spring Boot + PostgreSQL 프로젝트 초기화
- DB 스키마 + 시계열 파티셔닝
- KRX 크롤러 구현 (공매도 + 시세)
- Spring Batch Job 설정
- Next.js 프로젝트 초기화 + 레이아웃

### Sprint 2 (Week 3~4): 시그널 엔진 + 대시보드
- 3대 시그널 탐지 로직
- 시그널 API + 종목 시계열 API
- 시그널 리스트 페이지
- 종목 상세 차트 페이지 (Recharts)
- 면책 고지 컴포넌트

### Sprint 3 (Week 5~6): 백테스팅 + 알림 + 통합
- 백테스팅 엔진 + API
- 백테스팅 결과 페이지
- 텔레그램 Bot 연동 + 알림 스케줄링
- 통합 테스트 (커버리지 80%+)
- E2E 테스트
