## 218건 stock_name 빈 종목

### 요약
- `stock` 테이블 3,098건 중 218건의 `stock_name` = NULL/빈 문자열
- 3년 백필 + `scripts/fix_stock_names.py` 복구 후에도 매칭 안 된 코드
- **원인**: 현재 KRX `get_market_price_change_by_ticker(market='ALL')` 목록에 없는 종목 — 상장폐지/합병/티커 변경된 구코드로 추정
- 최근 1달(2026-04-) stock_price 거래 기록이 있는 건: **5건만** — 나머지 213건은 시계열 자체가 없음

### 분포 (prefix 기준)
| prefix | 건수 |
|---|---:|
| 010 | 5 |
| 450 | 5 |
| 001 | 4 |
| 446 | 4 |
| 000 | 4 |
| 430 | 4 |
| 442 | 4 |
| 448 | 4 |
| ... | ... |

상위 2~3자리 prefix가 분산되어 있어 특정 시장/섹터 편향은 없음. 주로 430~450 범위가 많은데 2023~2024 신규상장 후 합병·상장폐지 된 것으로 추정.

### UI 영향
- 포트폴리오 테이블: `h.stock_name ?? '(이름 없음)'` 으로 이미 null 처리됨 → 깨지지 않음
- 종목 상세(`/stocks/[code]`): 사용자가 직접 URL 입력 시에만 접근. `stock_code` 는 항상 있어 네비게이션 가능
- 시그널 대시보드: 해당 218건에 시그널이 거의 안 잡힘 (시계열 없음)

### 조치 결정
**현상 유지**. `is_active` 플래그 추가 마이그레이션은 ROI 낮음:
- UI 영향 사실상 없음 (null 대응 완료)
- 백필/탐지 경로에 영향 없음 (데이터 없으면 자연 제외)
- 실제 현 종목이 일부 섞여 있을 가능성 남겨둬야 추후 수동 수정 여지

### 재복구 방법
추후 과거 시점 KRX 데이터 확보 시:
```bash
# 과거 일자 기준으로 이름 매칭 시도 (예: 2023-06-01)
docker compose exec backend python -m scripts.fix_stock_names --date 2023-06-01
```
기존 fix_stock_names 는 빈 이름만 UPDATE 하므로 재실행 안전 (이미 채워진 2,880건 보존).
